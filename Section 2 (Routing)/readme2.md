## Metadata API in Next.js (s2 : ep7)

- how to set metadata of next js using metadata api.
- jo body de ander hota hai vo dikhta hai pr jo body ke ander nhi hota meta data head ke ander.
- we dont manually add head tag.
- we can use metadata api :
  - we export an object named metadata

```js
export const metadata = {
  title: "Create Next App",
  description: "Generated by create next app", // used for seo
};
```

- if this code on root layout then it will be same for every route
- i can also add it on other pages of routes.

```js

page.js inside root app folder

export const metadata = {
  title: "Home | Create Next App",
  description: "Generated by create next app",
};


page.js inside about folder

export const metadata = {
  title: "About | Next App",
  description: "Generated by create next app",
};

```

- we can also set template for this object in root layout.js.

```js

export const metadata = {
    title:{
        template:"%s | Technical agency"
        default: "Next App",
    }
  description: "Generated by create next app",
};

```

now here in

- template : %s we will recieve the title of other metadata objects.
- default : yhan basically jo value di hogi vo home page pe show hogi.

### generating dynamic title for every page.

```js
export async function generateMetadata({ params }) {
  const { blogId } = await params;

  return {
    absolute: `Blog ${blogID}`,
  };
}
```

- `i can also overwrite template object by absolute key`

```js
export const metadata = {
  title: "Files",
};
```

### Creating custom error page (404 s2: ep8)

- this error comes when page not exist
- notFound() : next js function to throw error.
- we can also render it on conditions
  ex: show error file when user go to blogs/test

```js
if (blog === "test") {
  return notFound();
}
// it will  check if the blog non string or not is string then throw error
if (!/^\d+$/.test(blog)) {
  return notFound();
}
```
- if we want to create a custom notFound page in nextjs then we can create a file name
    ` not-found.js` this file name necessary to be like this

```js

import React from "react";

export default () => {
  return (
    <div>
      <h1>This is my custom not found page.</h1>
    </div>
  );
};

```

we can also set header and footer diffrent for marketing and application iske liye layout.js ki copy dono folder me daal denge and root layout se header footer hta denge

```js

// application

export const metadata = {
  title: `application`,
};

export default function RootLayout({ children }) {
  return (
    <>
      <header className="bg-white/50 text-black">application header</header>
      <h1>Root layout</h1>
      {children}

      <footer className="bg-teal-500 text-black">application footer</footer>
    </>
  );
}


// marketing

export const metadata = {
  title: `Marketing`,
};

export default function RootLayout({ children }) {
  return (
    <>
      <header className="bg-white/50 text-black">Marketing header</header>
      <h1>Root layout</h1>
      {children}

      <footer className="bg-teal-500 text-black">Marketing footer</footer>
    </>
  );
}

```
